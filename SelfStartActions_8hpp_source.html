38 /// \ingroup TimeGroup
39 /// Definition of the integrator self-starting procedure.
40 ///
41 /// The self-start procedure generates \f$N\f$ function values of
42 /// accuracy \f$O\left((\Delta t)^N\right)\f$, where \f$N\f$ is the
43 /// negative of the initial slab number. To generate values, it
44 /// requires the time stepper to be a multistep integrator that will
45 /// produce an order-\f$k\f$-accurate result given \f$k-1\f$ history
46 /// values.
47 ///
48 /// If the integrator is started from analytic history data or
49 /// requires no history (such as for a substep integrator), then the
50 /// initial slab number can be set to zero and no self-start steps
51 /// will be taken.
52 ///
53 /// \details
54 /// To self-start a multistep integrator, the function is integrated
55 /// repeatedly with increasing accuracy. A first order integrator
56 /// (Euler's method) requires no history values, so it can be used,
57 /// starting from the initial point, to generate a
58 /// first-order-accurate value at a later time. We then reset to the
59 /// start conditions and use the new "history" value (at a discarded
60 /// point later in time) to take two steps with a second-order method.
61 /// These values are second-order-accurate despite the history only
62 /// being first-order because the calculation of the change in the
63 /// value multiplies the previous derivatives by a factor of
64 /// \f$\Delta t\f$. The time and value are then again reset to their
65 /// starting values and we start again at third order, and so on.
66 ///
67 /// The choice of performing the low-order integrations in the same
68 /// direction as the main integration makes this a _forward_
69 /// self-start procedure, as opposed to a _backward_ procedure that
70 /// produces values for times before the start time. The primary
71 /// advantage of the forward version is that the solution is
72 /// guaranteed to exist after the start time, but not before. It also
73 /// makes bookkeeping easier, as the reset after each order increase
74 /// is to the initial state, rather than to a time one step further
75 /// back for each order. It does have the disadvantage, however, of
76 /// leaving a non-monotonic history at the end of the procedure, which
77 /// the main evolution loop must be able to handle.
78 ///
79 /// Each time the state is reset the slab number is increased by one.
80 /// This ensures that the evaluations are considered to be ordered in
81 /// their evaluation order, even though they are not monotonic in
82 /// time. When the slab number reaches zero the initialization
83 /// procedure is complete and history appropriate for use for an
84 /// integrator of order \f$N+1\f$ has been generated.
85 ///
86 /// The self-start procedure performs all its evaluations before the
87 /// end of the first time step of the main evolution. It is important
88 /// that none of the early steps fall at the same time as the
89 /// self-start history values, so the main evolution should not
90 /// decrease its step size on the first step after the procedure.
91 /// Additionally, the history times will not be monotonicly increasing
92 /// until \f$N\f$ steps have been taken. The local-time-stepping
93 /// calculations require monotonic time, so local time-stepping should
94 /// not be initiated until the self-start values have expired from the
95 /// history. These restrictions on step-size changing are checked in
96 /// the TimeStepper::can_change_step_size method.

129 /// \ingroup ActionsGroup
130 /// \ingroup TimeGroup
131 /// Prepares the evolution for time-stepper self-starting.
132 ///
133 /// Stores the initial values of the variables and time step and sets
134 /// an appropriate step for self-starting.
135 ///
136 /// \details The self-start procedure must take place within one slab,
137 /// and we want to avoid the end of the slab so that we don't have to
138 /// deal with another action advancing the slab on us. There will be
139 /// problems if the main evolution tries to evaluate at a time that
140 /// the self-start procedure used before the self-start version falls
141 /// out of the history, so we have to make sure that does not happen.
142 /// We can't do that by making sure our steps are large enough to keep
143 /// ahead because of the slab restriction, so instead we have to make
144 /// the self-start step smaller to ensure no collisions. The easiest
145 /// way to do that is to fit the entire procedure before the first
146 /// real step, so we pick an initialization time step of
147 /// \f$\Delta t/(N+1)\f$, for \f$\Delta t\f$ the initial evolution
148 /// time step and \f$N\f$ the number of history points to be
149 /// generated.
150 ///
151 /// Uses:
152 /// - GlobalCache: nothing
153 /// - DataBox:
154 /// - Tags::TimeStepId
155 /// - Tags::TimeStep
156 /// - variables_tag
157 /// - primitive_variables_tag if the system has primitives
158 ///
159 /// DataBox changes:
160 /// - Adds:
161 /// - SelfStart::Tags::InitialValue<Tags::TimeStep>
162 /// - SelfStart::Tags::InitialValue<variables_tag>
163 /// - SelfStart::Tags::InitialValue<primitive_variables_tag> if the system
164 /// has primitives
165 /// - Removes: nothing
166 /// - Modifies: Tags::TimeStep
167 struct Initialize {
168  private:
169  template <typename TagsToSave>
170  struct StoreInitialValues;
171 
172  template <typename... TagsToSave>
173  struct StoreInitialValues<tmpl::list<TagsToSave...>> {
174  template <typename Box>
175  static auto apply(Box box) noexcept {
176  return db::create_from<
177  db::RemoveTags<>,
178  db::AddSimpleTags<Tags::InitialValue<TagsToSave>...>>(
179  std::move(box), std::make_tuple(db::get<TagsToSave>(box))...);
180  }
181  };
182 
183  public:
184  template <typename DbTags, typename... InboxTags, typename Metavariables,
185  typename ArrayIndex, typename ActionList,
186  typename ParallelComponent>
187  static auto apply(db::DataBox<DbTags>& box,
188  const tuples::TaggedTuple<InboxTags...>& /*inboxes*/,
189  const Parallel::GlobalCache<Metavariables>& /*cache*/,
190  const ArrayIndex& /*array_index*/,
191  const ActionList /*meta*/,
192  const ParallelComponent* const /*meta*/) noexcept {
193  using system = typename Metavariables::system;
194 
195  const TimeDelta initial_step = db::get<::Tags::TimeStep>(box);
196  // The slab number increments each time a new point is generated
197  // until it reaches zero.
198  const auto values_needed =
199  -db::get<::Tags::Next<::Tags::TimeStepId>>(box).slab_number();
200  TimeDelta self_start_step = initial_step;
201 
202  // Make sure the starting procedure fits in a slab.
203  if (abs(self_start_step * values_needed).fraction() >= 1) {
204  // Decrease the step so that the generated history will be
205  // entirely before the first step. This ensures we will not
206  // generate any duplicate times in the history as we start the
207  // real evolution.
208  self_start_step /= (values_needed + 1);
209  }
210 
211  auto new_box = StoreInitialValues<tmpl::push_back<
212  detail::vars_to_save<system>, ::Tags::TimeStep>>::apply(std::move(box));
213 
214  db::mutate<::Tags::TimeStep>(
215  make_not_null(&new_box), [&self_start_step](
216  const gsl::not_null<::TimeDelta*>
217  time_step) noexcept {
218  *time_step = self_start_step;
219  });
220 
221  return std::make_tuple(std::move(new_box));
222  }
223 };

227 /// Exits the self-start loop if the required order has been reached.
228 ///
229 /// Uses:
230 /// - GlobalCache: nothing

234 /// - Adds: nothing
235 /// - Removes: nothing
236 /// - Modifies: nothing
237 template <typename ExitTag>
238 struct CheckForCompletion {
239  template <typename DbTags, typename... InboxTags, typename Metavariables,
240  typename ArrayIndex, typename ActionList,
241  typename ParallelComponent>
242  static std::tuple<db::DataBox<DbTags>&&, bool, size_t> apply(
243  db::DataBox<DbTags>& box,
244  const tuples::TaggedTuple<InboxTags...>& /*inboxes*/,
245  const Parallel::GlobalCache<Metavariables>& /*cache*/,

248  // The self start procedure begins with slab number
249  // -number_of_past_steps and counts up. When we reach 0 we should

252  // next time should be the first real step.

254  return {std::move(box), false,
255  tmpl::index_of<ActionList, ::Actions::Label<ExitTag>>::value};
256  }
257  return {std::move(box), false,
258  tmpl::index_of<ActionList, CheckForCompletion>::value + 1};
259  }

262 /// \ingroup ActionsGroup
263 /// \ingroup TimeGroup
264 /// If we have taken enough steps for this order, set the next time to
265 /// the start time and increment the slab number
266 ///
267 /// Uses:
268 /// - GlobalCache: nothing
269 /// - DataBox:
270 /// - Tags::HistoryEvolvedVariables
271 /// - Tags::SubstepTime
272 /// - Tags::TimeStepId
273 /// - Tags::TimeStep
274 ///
275 /// DataBox changes:
276 /// - Adds: nothing
277 /// - Removes: nothing
278 /// - Modifies: Tags::Next<Tags::TimeStepId> if there is an order increase
279 struct CheckForOrderIncrease {
280  template <typename DbTags, typename... InboxTags, typename Metavariables,
281  typename ArrayIndex, typename ActionList,
282  typename ParallelComponent>
283  static std::tuple<db::DataBox<DbTags>&&> apply(
284  db::DataBox<DbTags>& box,
285  const tuples::TaggedTuple<InboxTags...>& /*inboxes*/,
286  const Parallel::GlobalCache<Metavariables>& /*cache*/,
287  const ArrayIndex& /*array_index*/, ActionList /*meta*/,
288  const ParallelComponent* const /*meta*/) noexcept { // NOLINT const
289  const auto& time = db::get<::Tags::SubstepTime>(box);
290  const auto& time_step = db::get<::Tags::TimeStep>(box);
291  const auto& history = db::get<::Tags::HistoryEvolvedVariables<>>(box);
292 
293  const Time required_time =
294  (time_step.is_positive() ? time.slab().start() : time.slab().end()) +
295  (history.size() + 1) * time_step;
296  const bool done_with_order = time == required_time;
297 
298  if (done_with_order) {
299  db::mutate<::Tags::Next<::Tags::TimeStepId>>(
300  make_not_null(&box),
301  [](const gsl::not_null<::TimeStepId*> next_time_id,
302  const ::TimeStepId& current_time_id) noexcept {
303  const Slab slab = current_time_id.step_time().slab();
304  *next_time_id =
305  TimeStepId(current_time_id.time_runs_forward(),
306  current_time_id.slab_number() + 1,
307  current_time_id.time_runs_forward() ? slab.start()
308  : slab.end());
309  },
310  db::get<::Tags::TimeStepId>(box));
311  }
312 
313  return {std::move(box)};
314  }
315 };

317 /// \ingroup ActionsGroup
318 /// \ingroup TimeGroup
319 /// Jumps to the start of the self-start algorithm (skipping taking a
320 /// step from the last point) if the generation of the points for the
321 /// current order is complete.
322 ///
323 /// Uses:
324 /// - GlobalCache: nothing
325 /// - DataBox:
326 /// - Tags::Next<Tags::TimeStepId>
327 /// - SelfStart::Tags::InitialValue<variables_tag>
328 /// - SelfStart::Tags::InitialValue<primitive_variables_tag> if the system
329 /// has primitives
330 ///
331 /// DataBox changes:
332 /// - Adds: nothing
333 /// - Removes: nothing
334 /// - Modifies:
335 /// - variables_tag if there is an order increase
336 /// - primitive_variables_tag if there is an order increase and the system
337 /// has primitives
338 template <typename RestartTag>
339 struct StartNextOrderIfReady {
340  template <typename DbTags, typename... InboxTags, typename Metavariables,
341  typename ArrayIndex, typename ActionList,
342  typename ParallelComponent>
343  static std::tuple<db::DataBox<DbTags>&&, bool, size_t> apply(
344  db::DataBox<DbTags>& box,
345  const tuples::TaggedTuple<InboxTags...>& /*inboxes*/,
346  const Parallel::GlobalCache<Metavariables>& /*cache*/,
347  const ArrayIndex& /*array_index*/, const ActionList /*meta*/,
348  const ParallelComponent* const /*meta*/) noexcept {
349  using system = typename Metavariables::system;
350 
351  constexpr size_t restart_index =
352  tmpl::index_of<ActionList, ::Actions::Label<RestartTag>>::value + 1;
353  constexpr size_t continue_index =
354  tmpl::index_of<ActionList, StartNextOrderIfReady>::value + 1;
355 
356  const bool done_with_order =
357  db::get<::Tags::Next<::Tags::TimeStepId>>(box).is_at_slab_boundary();
358 
359  if (done_with_order) {
360  tmpl::for_each<detail::vars_to_save<system>>([&box](auto tag) noexcept {
361  using Tag = tmpl::type_from<decltype(tag)>;
362  db::mutate<Tag>(
363  make_not_null(&box),
364  [](const gsl::not_null<typename Tag::type*> value,
365  const std::tuple<typename Tag::type>& initial_value) noexcept {
366  *value = get<0>(initial_value);
367  },
368  db::get<Tags::InitialValue<Tag>>(box));
369  });
370  }
371 
372  return {std::move(box), false,
373  done_with_order ? restart_index : continue_index};
374  }
375 };
