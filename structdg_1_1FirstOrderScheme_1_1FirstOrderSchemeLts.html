Detailed Description

template<size_t Dim, typename VariablesTag, typename DtVariablesTag, typename NumericalFluxComputerTag, typename TemporalIdTag, typename TimeStepperTag>
struct dg::FirstOrderScheme::FirstOrderSchemeLts< Dim, VariablesTag, DtVariablesTag, NumericalFluxComputerTag, TemporalIdTag, TimeStepperTag >

Boundary contributions for a first-order DG scheme with local time-stepping.

This class is the local time-stepping equivalent to the dg::FirstOrderScheme::FirstOrderScheme. Notable differences are:

  • Boundary contributions are added to the VariablesTag directly.
  • We use the Tags::BoundaryHistory on mortars.
  • We need to store the face-normal magnitude in the boundary history, as opposed to the non-LTS case where we can retrieve it when needed.
