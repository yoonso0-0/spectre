      97             : /// the reader should refer to the reference for full details. The limiter has a
      98             : /// general implementation that can work on an arbitrary set of tensors; the
      99             : /// limiting algorithm is applied to each component of each tensor
     100             : /// independently.
     101             : ///
     102             : /// The MUSCL and \f$\Lambda\Pi^1\f$ limiters are both intended for use on
     103             : /// piecewise-linear solutions, i.e., on linear-order elements with two points
     104             : /// per dimension. These limiters operate by reducing the spatial slope of the
     105             : /// tensor components if the data look like they may contain oscillations.
     106             : /// Between these two, MUSCL is more dissipative --- it more aggressively
     107             : /// reduces the slopes of the data, so it may better handle strong shocks, but
     108             : /// it correspondingly produces the most broadening of features in the solution.
     109             : ///
     110             : /// Note that we do not _require_ the MUSCL and \f$\Lambda\Pi^1\f$ limiters to
     111             : /// be used on linear-order elements. However, when they are used on a
     112             : /// higher-resolution grid, the limiters act to linearize the solution (by
     113             : /// discarding higher-order mode content) whether or not the slopes must be
     114             : /// reduced.
     115             : ///
     116             : /// The \f$\Lambda\Pi^N\f$ limiter is intended for use with higher-order
     117             : /// elements (with more than two points per dimension), where the solution is a
     118             : /// piecewise polynomial of higher-than-linear order. This limiter generalizes
     119             : /// \f$\Lambda\Pi^1\f$: the post-limiter solution is the linearized solution of
     120             : /// \f$\Lambda\Pi^1\f$ in the case that the slopes must be reduced, but is the
     121             : /// original (higher-order) data in the case that the slopes are acceptable.
     122             : ///
     123             : /// For all three types of minmod limiter, the algorithm can be relaxed from
     124             : /// TVD (total variation diminishing) in the means to TVB (total variation
     125             : /// bound) in the means. This may avoid limiting away smooth extrema in the
     126             : /// solution that would otherwise look like spurious oscillations. When this
     127             : /// correction is enabled, the limiter will not reduce the slope (but may still
     128             : /// linearize) on elements where the slope is less than \f$m h^2\f$, where
     129             : /// \f$m\f$ is the TVB constant and \f$h\f$ is the size of the DG element.
     130             : /// Note the "in the means" qualifier: the limiter controls the oscillation
     131             : /// between the mean solution values across neighboring cells, but may not
     132             : /// control oscillations within the cells.
     133             : ///
     134             : /// The choice of the TVB constant \f$m\f$ is difficult. Larger values result in
     135             : /// fewer limiter activations, especially near smooth extrema in the solution
     136             : /// --- this can help to avoid incorrectly limiting away these smooth extrema,
     137             : /// but can also result in insufficient limiting of truly spurious oscillations.
     138             : /// The reference uses a value of 50 when presenting the limiter with simple
     139             : /// shock tests, but in general the value \f$m\f$ that optimizes between
     140             : /// robustness and minimal loss of accuracy is problem dependent.
     141             : ///
     142             : /// The limiter acts in the `Frame::Logical` coordinates, because in these
     143             : /// coordinates it is straightforward to formulate the algorithm. This means the
     144             : /// limiter can operate on generic deformed grids. However, if the grid is too
     145             : /// strongly deformed, some things can start to break down:
     146             : /// 1. When an element is deformed so that the Jacobian (from `Frame::Logical`
     147             : ///    to `Frame::Inertial`) varies across the element, then the limiter fails
     148             : ///    to be conservative. In other words, the integral of a tensor `u` over the
     149             : ///    element will change after the limiter activates on `u`. This error is
     150             : ///    typically small.
     151             : /// 2. When there is a sudden change in the size of the elements (perhaps at an
     152             : ///    h-refinement boundary, or at the boundary between two blocks with very
     153             : ///    different mappings), a smooth solution in `Frame::Inertial` can appear
     154             : ///    to have a kink in `Frame::Logical`. The Minmod implementation includes
