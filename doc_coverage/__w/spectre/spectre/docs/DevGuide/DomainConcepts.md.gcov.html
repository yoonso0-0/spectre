       5             : # Domain Concepts {#domain_concepts}
       6             : 
       7             : * Logical Coordinates:<br>
       8             :   The coordinate frame of a reference cell.
       9             : 
      10             : * Domain:<br>
      11             :   The computational domain is partitioned into cells called Blocks. Furthermore, each Block
      12             :   must have at most one neighboring Block in each Direction.
      13             : 
      14             : * Block:<br>
      15             :   A reference cell embedded into a subset of the computational domain. Blocks
      16             :   are identified by unique integral values.
      17             : 
      18             : * Orientation:<br>
      19             :   The information of how the Logical Coordinates between Blocks are related.
      20             : 
      21             : * \ref BlockNeighbor "Block Neighbor":<br>
      22             :   The identity and Orientation of a neighbor of a given Block.
      23             : 
      24             : * Direction:<br>
      25             :   A logical coordinate axis and a label "Upper" or "Lower" depending on whether
      26             :   the Direction is aligned with or anti-aligned with the logical coordinate
      27             :   axis, respectively.
      28             : 
      29             : * Refinement Level:<br>
      30             :   The number of times a Block is split in half in a given dimension.
      31             : 
      32             : * Segment:<br>
      33             :   The specific subset of the interval \f$[-1, 1]\f$ defined by the Refinement Level and an integer label
      34             :   such that the Segment's interval is \f$[-1 + 2*(N/D), -1 + 2*(N+1)/D]\f$ where N is the integer label and
      35             :   D is \f$2^{(\textrm{Refinement Level})}\f$.
      36             : 
      37             : * \ref SegmentId "Segment Identifier":<br>
      38             :   The Refinement Level and an integer labeling a Segment.
      39             : 
      40             : * Element:<br>
      41             :   A refined subregion of a Block defined by its Segments in each
      42             :   logical coordinate. The properties of the Element
      43             :   are inherited from the block, i.e. it is self-similar to the Block.
      44             : 
      45             : * \ref ElementId "Element Identifier":<br>
      46             :   The Block Identifier containing the Element and a Segment Identifier
      47             :   in each logical coordinate axis.
      48             : 
      49             : * Mesh:<br>
      50             :   A regular set of grid points associated with an Element. Represented by the data structure Mesh.
      51             : 
      52             : * External Boundary:<br>
      53             :   A face of a Block or Element that has no neighbor.
      54             : 
      55             : * Internal Boundary:<br>
      56             :   A boundary that is not an External Boundary.
      57             : 
      58             : * \ref Neighbors "Face Neighbors" (Rename C++ Neighbors class to FaceNeighbors):<br>
      59             :   The identities and Orientation of the neighbors in a particular direction of a given Element.
      60             : 
      61             : * External Boundary Condition:<br>
      62             :   A prescription for updating the solution on an External Boundary. Each
      63             :   External Boundary of Block has exactly one External Boundary Condition
      64             :   associated with it.
