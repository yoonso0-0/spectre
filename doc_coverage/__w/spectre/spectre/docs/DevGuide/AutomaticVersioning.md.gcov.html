      13             : # Manually creating releases
      14             : 
      15             : The GitHub workflow responsible for automatic versioning also allows creating
      16             : a release manually. To create a release, follow the instructions to manually
      17             : run a workflow:
      18             : 
      19             : - [Manually running a workflow on GitHub](https://docs.github.com/actions/managing-workflow-runs/manually-running-a-workflow)
      20             : 
      21             : To create a release you will have to run the workflow "Tests" on the `develop`
      22             : branch _and_ type in a valid release version name. The workflow will only create
      23             : the release if the version name matches the format defined above and if it
      24             : matches the date at the time the "Release version" job runs.

      26             : # Release notes
      27             : 
      28             : The release notes are compiled automatically based on the activity in the
      29             : repository since the last release. They will contain a list of merged
      30             : pull-requests. Pull-requests labeled "major new feature" or "bugfix" on GitHub
      31             : will be classified as such in the release notes.
      32             : 
      33             : The script `tools/CompileReleaseNotes.py` generates the release notes and can
      34             : also be invoked manually with a Python 3 interpreter to retrieve an overview of
      35             : what happened in the repository recently. The script requires you install
      36             : `GitPython`, `PyGithub` and `tqdm` in your Python environment.
