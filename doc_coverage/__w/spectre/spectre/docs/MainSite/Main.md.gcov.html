      65             : # What is SpECTRE? {#intro_sec}
      66             : 
      67             : SpECTRE is an open-source code for multi-scale, multi-physics problems
      68             : in astrophysics and gravitational physics. In the future, we hope that
      69             : it can be applied to problems across discipline boundaries in fluid
      70             : dynamics, geoscience, plasma physics, nuclear physics, and
      71             : engineering. It runs at petascale and is designed for future exascale
      72             : computers.
      73             : 
      74             : SpECTRE is being developed in support of our collaborative Simulating
      75             : eXtreme Spacetimes (SXS) research program into the multi-messenger
      76             : astrophysics of neutron star mergers, core-collapse supernovae, and
      77             : gamma-ray bursts.
      78             : 
      79             : ## Citing SpECTRE
      80             : 
      81             : Please cite SpECTRE in any publications that make use of its code or data. Cite
      82             : the latest version that you use in your publication. The DOI for this version
      83             : is:
      84             : 
      85             : - DOI: [\spectredoi](https://doi.org/\spectredoi)
      86             : 
      87             : You can cite this BibTeX entry in your publication:
      88             : 
      89             : - [Find BibTeX entry for this version on Zenodo](https://zenodo.org/record/\spectrezenodoid/export/hx)
      90             : 
      91             : To aid reproducibility of your scientific results with SpECTRE, we recommend you
      92             : keep track of the version(s) you used and report this information in your
      93             : publication. We also recommend you supply the YAML input files and, if
      94             : appropriate, any additional C++ code you wrote to compile SpECTRE executables as
      95             : supplemental material to the publication.
      96             : 
      97             : ## Navigating the Documentation {#navigate_documentation_sec}
      98             : 
      99             : The SpECTRE documentation is organized into tutorials, developer guides, groups
     100             : of related code, namespaces, and files for easier navigation. These can all be
     101             : accessed by links in the menu bar at the top.
     102             : 
     103             : - For instructions on **installing SpECTRE** on personal computers and clusters
     104             :   consult the \ref installation "Installation" and \ref installation_on_clusters
     105             :   "Installation on clusters" pages, respectively.
     106             : - If you are looking to **run simulations with SpECTRE** we recommend starting
     107             :   with the \ref tutorials "User Tutorials". The tutorials are designed to get
     108             :   users up and running with a simulation, as well as analyzing and visualizing
     109             :   the output.
     110             : - For people looking to **contribute to SpECTRE** there are tutorials on the
     111             :   \ref dev_guide "Dev Guide" page. For instance, the dev guide details the \ref
     112             :   code_review_guide "code review guidelines", how to \ref writing_unit_tests
     113             :   "write unit tests", how to \ref writing_good_dox "write documentation", and
     114             :   also provides information about C++ and the philosophy behind SpECTRE
     115             :   development.
     116             : - The [Topics](modules.html) section under the _Reference_ menu item contains
     117             :   groups of related code (managed through doxygen groups). For example, there is
     118             :   a group for all the data structures we use, a group for utility functions and
     119             :   classes, as well as groups for coordinate maps, domain creation, and many
     120             :   others. The [Topics](modules.html) are designed to help developers discover
     121             :   existing functionality so that things are not re-implemented several times.
     122             : - You can also get an overview of the code base by namespace by visiting the
     123             :   [Namespaces](namespaces.html) section under the _Reference_ menu item.
     124             : - Finally, it is also possible to browse the repository by files under the
     125             :   [Files](files.html) menu item, though it is recommended that in that case you
     126             :   browse the [GitHub repository](https://github.com/sxs-collaboration/spectre)
     127             :   directly.
