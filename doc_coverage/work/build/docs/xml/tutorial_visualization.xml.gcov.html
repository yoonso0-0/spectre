      45             : <title>3D %Data Volume %Data In ParaView</title>
      46             : <para>A SpECTRE executable with observers produces volume and/or reduced data <ref refid="namespaceh5" kindref="compound">h5</ref> files. An XDMF file must be created from the volume data in order to do visualization using ParaView. To this end we provide the python executable <computeroutput>GenerateXdmf.py</computeroutput> in the <computeroutput>Visualization/Python</computeroutput> directory. <computeroutput>GenerateXdmf.py</computeroutput> takes two arguments which are passed to <computeroutput>--file-prefix</computeroutput> and <computeroutput>--output</computeroutput>. The argument passed to <computeroutput>--file-prefix</computeroutput> is the name of the H5 volume data, leaving out the node number and extension. The argument passed to <computeroutput>--output</computeroutput> is the desired .xmf file name, also without filename extension. Use <computeroutput>--help</computeroutput> to see a further description of possible arguments.</para>
      47             : <para>Open the .xmf file in ParaView and select the <computeroutput>Xdmf Reader</computeroutput>, <emphasis>not</emphasis> the version 3 readers. On the left hand side of the main ParaView window is a section named <computeroutput>Properties</computeroutput>, here you must click the highlighted <computeroutput>Apply</computeroutput> button. ParaView will now render your volume data. If you only wish to visualize a few datasets out of a large set, we recommended unchecking the boxes for the datasets you wish to ignore under <computeroutput>Point Arrays</computeroutput> before clicking <computeroutput>Apply</computeroutput>.</para>

      58             : <para>New volume data can be created from the existing volume data using the <computeroutput>Calculator</computeroutput> filter. In the <computeroutput>Calculator</computeroutput>&apos;s text box, input a numerical expression in terms of existing datasets evaluating to the desired quantity. By default, this will create a new dataset <computeroutput>Result</computeroutput>.       65             : <sect2 id="tutorial_visualization_1autotoc_md204">
      66             : <title>Reproducibility of Results</title>
      67             : <para>Being able to reproduce the results of simulations is important for scientific integrity and accuracy. To this end, SpECTRE encodes the source tree, environment variables, and a record of the third party library versions into the executables. From most executables these can be retrieved using the flags <computeroutput>--dump-source-as filename_without_extension</computeroutput>, <computeroutput>--dump-paths</computeroutput>, <computeroutput>--dump-library-version</computeroutput>, and <computeroutput>--dump-environment</computeroutput>. The flag <computeroutput>--dump-only</computeroutput> will cause the program to exit after the information has been dumped. You can use the <computeroutput>--help</computeroutput> flag to get more details on what flags are available. The flag <computeroutput>--dump-paths</computeroutput> will print to screen the environment variables <computeroutput>PATH</computeroutput>, <computeroutput>CPATH</computeroutput>, <computeroutput>LD_LIBRARY_PATH</computeroutput>, <computeroutput>LIBRARY_PATH</computeroutput>, and <computeroutput>CMAKE_PREFIX_PATH</computeroutput> at link time. The flag <computeroutput>--dump-library-versions</computeroutput> will print to screen the contents of the <computeroutput>LibraryVersions.txt</computeroutput> file generated by CMake. The flag <computeroutput>--dump-environment</computeroutput> will print to screen the output of <computeroutput>printenv</computeroutput> at link time. Finally, the flag <computeroutput>--dump-source-as filename_without_extension</computeroutput> will dump the archive <computeroutput>filename_without_extension.tar.gz</computeroutput>, which contains the entire source tree at link time. The combined above information should be enough to reproduce simulations results to roundoff error as long as the third party libraries used to build the executable have not changed on the system the simulation was run on.</para>
      68             : <para>Often only data from the simulation is retained, not the entire executable. In order to make simulations reproducible from only the HDF5 output, any HDF5 files written with SpECTRE&apos;s HDF5 wrappers contain the info about the run, output of <computeroutput>printenv</computeroutput> at link time, the contents of <computeroutput>LibraryVersions.txt</computeroutput>, and an archive of the source tree. These are inside the <computeroutput>header.hdr</computeroutput> group with names <computeroutput>SimulationInfo</computeroutput>, <computeroutput>EnvironmentVariablesAtCompilation</computeroutput>, <computeroutput>LibraryVersions</computeroutput>, and <computeroutput>src.tar.gz</computeroutput>. The source archive can be extracted by running the command <programlisting><codeline><highlight class="normal">h5dump<sp/>-d<sp/>/src.tar.gz<sp/>-b<sp/>LE<sp/>-o<sp/>src.tar.gz<sp/>/path/to/hdf5/file.h5</highlight></codeline>
      69             : </programlisting></para>
      70             : <para>You will need to make sure the argument to <computeroutput>-b</computeroutput> is the same endianness as your computer. In the above example little endian is used. </para>
