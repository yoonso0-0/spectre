     191             : <title>Behind the scenes:  the main parallel component</title>
     192             : <para>Every Charm++ executable needs a mainchare. All SpECTRE executables use <computeroutput>Main</computeroutput> (found in <computeroutput><ref refid="Main_8hpp" kindref="compound">src/Parallel/Main.hpp</ref></computeroutput>) as the mainchare. When a SpECTRE executable is run, the constructor of <computeroutput>Main</computeroutput> that is executed takes the command line options as an argument (as type <computeroutput>CkArgMsg*</computeroutput>). This constructor performs the following operations:</para>
     193             : <para><itemizedlist>
     194             : <listitem><para>Prints useful startup information</para>
     195             : </listitem><listitem><para>Parses the command line options, performing any requested operations</para>
     196             : </listitem><listitem><para>Parses the input file options, populating a tagged tuple with their values. (This is discussed in more detail in a future tutorial on input file options.)</para>
     197             : </listitem><listitem><para>Creates the <computeroutput>GlobalCache</computeroutput> (a nodegroup chare) that holds objects created from input file options that are stored once per Charm++ node, as well as proxies to all other parallel components.</para>
     198             : </listitem><listitem><para>Creates user-requested non-array parallel components, passing them the proxy to the <computeroutput>GlobalCache</computeroutput> as well as any items they request that can be created from input file options.</para>
     199             : </listitem><listitem><para>Creates empty array user-requested parallel components</para>
     200             : </listitem><listitem><para>Sends the complete list of parallel components to the <computeroutput>GlobalCache</computeroutput>.</para>
     201             : </listitem></itemizedlist>
     202             : </para>
     203             : <para>Once the list of parallel components is sent to the <computeroutput>GlobalCache</computeroutput> on each node, the <computeroutput>Main</computeroutput> member function <computeroutput>allocate_array_components_and_execute_initial_phase</computeroutput> will be executed. This will allocate the elements of the array parallel components by calling the <computeroutput>allocate_array</computeroutput> member function of each component. Then the <computeroutput>start_phase</computeroutput> member function is called on each component, which will execute the phase action list for the <computeroutput>Initialization</computeroutput> phase for each component. Charm++ will execute each phase until it detects that nothing is happening (quiescence detection). As this represents a global synchronization point, the number of phases should be minimized in order to exploit the power of SpECTRE. After each phase, the <computeroutput>execute_next_phase</computeroutput> member function of <computeroutput>Main</computeroutput> will be called. This member function first determines what the next phase is. If the next phase is <computeroutput>Exit</computeroutput>, then some useful information is printed and the program exits gracefully. Otherwise the <computeroutput>execute_next_phase</computeroutput> member function of each parallel component is called. </para>
