      39             : /// Definition of the integrator self-starting procedure.
      40             : ///
      41             : /// The self-start procedure generates \f$N\f$ function values of
      42             : /// accuracy \f$O\left((\Delta t)^N\right)\f$, where \f$N\f$ is the
      43             : /// negative of the initial slab number.  To generate values, it
      44             : /// requires the time stepper to be a multistep integrator that will
      45             : /// produce an order-\f$k\f$-accurate result given \f$k-1\f$ history
      46             : /// values.
      47             : ///
      48             : /// If the integrator is started from analytic history data or
      49             : /// requires no history (such as for a substep integrator), then the
      50             : /// initial slab number can be set to zero and no self-start steps
      51             : /// will be taken.
      52             : ///
      53             : /// \details
      54             : /// To self-start a multistep integrator, the function is integrated
      55             : /// repeatedly with increasing accuracy.  A first order integrator
      56             : /// (Euler's method) requires no history values, so it can be used,
      57             : /// starting from the initial point, to generate a
      58             : /// first-order-accurate value at a later time.  We then reset to the
      59             : /// start conditions and use the new "history" value (at a discarded
      60             : /// point later in time) to take two steps with a second-order method.
      61             : /// These values are second-order-accurate despite the history only
      62             : /// being first-order because the calculation of the change in the
      63             : /// value multiplies the previous derivatives by a factor of
      64             : /// \f$\Delta t\f$.  The time and value are then again reset to their
      65             : /// starting values and we start again at third order, and so on.
      66             : ///
      67             : /// The choice of performing the low-order integrations in the same
      68             : /// direction as the main integration makes this a _forward_
      69             : /// self-start procedure, as opposed to a _backward_ procedure that
      70             : /// produces values for times before the start time.  The primary
      71             : /// advantage of the forward version is that the solution is
      72             : /// guaranteed to exist after the start time, but not before.  It also
      73             : /// makes bookkeeping easier, as the reset after each order increase
      74             : /// is to the initial state, rather than to a time one step further
      75             : /// back for each order.  It does have the disadvantage, however, of
      76             : /// leaving a non-monotonic history at the end of the procedure, which
      77             : /// the main evolution loop must be able to handle.
      78             : ///
      79             : /// Each time the state is reset the slab number is increased by one.
      80             : /// This ensures that the evaluations are considered to be ordered in
      81             : /// their evaluation order, even though they are not monotonic in
      82             : /// time.  When the slab number reaches zero the initialization
      83             : /// procedure is complete and history appropriate for use for an
      84             : /// integrator of order \f$N+1\f$ has been generated.
      85             : ///
      86             : /// The self-start procedure performs all its evaluations before the
      87             : /// end of the first time step of the main evolution.  It is important
      88             : /// that none of the early steps fall at the same time as the
      89             : /// self-start history values, so the main evolution should not
      90             : /// decrease its step size on the first step after the procedure.
      91             : /// Additionally, the history times will not be monotonicly increasing
      92             : /// until \f$N\f$ steps have been taken.  The local-time-stepping
      93             : /// calculations require monotonic time, so local time-stepping should
      94             : /// not be initiated until the self-start values have expired from the
      95             : /// history.  These restrictions on step-size changing are checked in
      96             : /// the TimeStepper::can_change_step_size method.
