26 /*!
27  * \ingroup HDF5Group
28  * \brief A volume data subfile written inside an H5 file.
29  *
30  * The volume data inside the subfile can be of any dimensionality greater than
31  * zero. This means that in a 3D simulation, data on 2-dimensional surfaces are
32  * written as a VolumeData subfile. Data can be written using the
33  * `write_volume_data()` method. An integral observation id is used to keep
34  * track of the observation instance at which the data is written, and
35  * associated with it is a floating point observation value, such as the
36  * simulation time at which the data was written. The observation id will
37  * generally be the result of hashing the temporal identifier used for the
38  * simulation.
39  *
40  * The data stored in the subfile are the tensor components passed to the
41  * `write_volume_data()` method as a `std::vector<ExtentsAndTensorVolumeData>`.
42  * The name of each tensor component must follow the format
43  * `GRID_NAME/TENSOR_NAME_COMPONENT`, e.g. `Element0/T_xx`. Typically the
44  * `GRID_NAME` should be the output of the stream operator of the spatial ID of
45  * the parallel component element sending the data to be observed. For example,
46  * in the case of a dG evolution where the spatial IDs are `ElementId`s, the
47  * grid names would be of the form `[B0,(L2I3,L2I3,L2I3)]`. The data are
48  * written contiguously inside of the H5 subfile, in that each tensor
49  * component has a single dataset which holds all of the data from
50  * all elements, e.g. a tensor component `T_xx` which is found on all grids
51  * appears in the path `H5_FILE_NAME/element_data.vol/T_xx` and that is where
52  * all of the `T_xx` data from all of the grids resides. Note that coordinates
53  * must be written as tensors in order to visualize the data in ParaView,
54  * Visit, etc. In order to reconstruct which data came from which grid,
55  * the `get_grid_names()`, and `get_extents()` methods list the grids
56  * and their extents in the order which they and the data were written.
57  * For example, if the first grid has name `GRID_NAME` with extents
58  * `{2, 2, 2}`, it was responsible for contributing the first 2*2*2 = 8 grid
59  * points worth of data in each tensor dataset. Use the
60  * `h5::offset_and_length_for_grid` function to compute the offset into the
61  * contiguous dataset that corresponds to a particular grid.
62  *
63  * \warning Currently the topology of the grids is assumed to be tensor products
64  * of lines, i.e. lines, quadrilaterals, and hexahedrons. However, this can be
65  * extended in the future. If support for more topologies is required, please
66  * file an issue.
67  */

111  /// Find the observation ID that matches the `observation_value`
112  size_t find_observation_id(const double observation_value) const noexcept {
113  for (auto& observation_id : list_observation_ids()) {
114  if (get_observation_value(observation_id) == observation_value) {
115  return observation_id;

118  ERROR("No observation with value " << observation_value

165 /*!
166  * \brief Find the interval within the contiguous dataset stored in
167  * `h5::VolumeData` that holds data for a particular `grid_name`.
168  *
169  * `h5::VolumeData` stores data for all grids that compose the volume
170  * contiguously. This function helps with reconstructing which part of that
171  * contiguous dataset belongs to a particular grid. See the `h5::VolumeData`
172  * documentation for more information on how it stores data.
173  *
174  * To use this function, call `h5::VolumeData::get_grid_names` and
175  * `h5::VolumeData::get_extents` and pass the results for the `all_grid_names`
176  * and `all_extents` arguments, respectively. This means you can retrieve this
177  * information from an `h5::VolumeData` once and use it to call
178  * `offset_and_length_for_grid` multiple times with different `grid_name`s.
179  *
180  * Here is an example for using this function:
181  *
182  * \snippet Test_VolumeData.cpp find_offset
183  *
184  * \see `h5::VolumeData`
185  */
186 std::pair<size_t, size_t> offset_and_length_for_grid(
187  const std::string& grid_name,
188  const std::vector<std::string>& all_grid_names,
189  const std::vector<std::vector<size_t>>& all_extents) noexcept;
